# UI 工作流：UI 原型图提示词生成

本工作流将激活"UI 提示词设计师"技能，解读产品规格文档并生成高质量的图像生成提示词。

## 前置条件

- ✅ 必须存在 `_Product-Spec.md` 文件

## 执行步骤

1. **读取产品规格文档**

   - 自动读取 `_Product-Spec.md`
   - 提取关键视觉元素和功能描述

2. **激活 UI 提示词设计师技能**

   - 使用 `ui-prompt-designer` 技能
   - 分析每个核心页面的布局需求

3. **逐页生成提示词**
   对于每个关键页面：

   - 确认页面类型（登录页、仪表盘、列表页等）
   - 明确必须显示的文字内容
   - 定义配色方案和视觉风格
   - 生成结构化的 Prompt

4. **创建 UI-Prompts.md**
   在项目根目录创建 `UI-Prompts.md`，格式如下：

   ```markdown
   # UI 原型图提示词集合

   ## 页面 1：[页面名称]

   ### 提示词
   ```

   [完整的图像生成提示词]

   ```

   ### 设计规范
   - 配色：#主色 / #辅助色
   - 字体：字体名称
   - 布局：描述

   ### 生成的原型图
   ![页面1原型](./assets/页面1.png)

   ---

   ## 页面 2：[页面名称]
   ...
   ```

5. **生成原型图**

   - 使用 Nano Banana Pro 逐个生成原型图
   - 保存到 `./assets/` 目录
   - 在文档中嵌入图片引用

6. **输出设计规范**
   - 汇总配色方案
   - 汇总字体规范
   - 汇总组件尺寸

## 输出物

- `UI-Prompts.md`：包含所有页面的提示词和原型图
- `./assets/`：原型图图片文件
