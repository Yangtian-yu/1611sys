# PRD 工作流：产品需求文档生成

本工作流将激活"毒舌产品经理"技能，通过深度追问收集需求，并生成结构化的产品规格文档。

## 执行步骤

1. **激活毒舌产品经理技能**

   - 使用 `ruthless-pm` 技能进行需求收集
   - 开启"灵魂拷问"模式，深挖用户真实需求

2. **需求收集清单**
   询问以下核心问题：

   - 目标用户是谁？（具体画像）
   - 核心痛点是什么？
   - 现有解决方案有哪些？为什么不够好？
   - 核心功能优先级排序（P0/P1/P2）
   - 技术约束和非功能性需求

3. **生成 Product-Spec.md**
   在项目根目录创建 `_Product-Spec.md`，包含以下章节：

   ```markdown
   # 产品规格文档

   ## 项目概述

   - 项目名称
   - 目标用户
   - 核心价值主张

   ## 功能需求

   ### P0 功能（必须有）

   ### P1 功能（应该有）

   ### P2 功能（可以有）

   ## 用户故事

   - 作为 [角色]，我想要 [功能]，以便 [目标]

   ## 技术约束

   - 性能要求
   - 安全要求
   - 兼容性要求

   ## 验收标准

   - 可量化的成功指标

   ## 风险与备选方案
   ```

4. **确认与迭代**
   - 向用户展示生成的文档
   - 根据反馈进行修改
   - 确认无误后标记为"已完成"

## 输出物

- `_Product-Spec.md`：完整的产品规格文档
