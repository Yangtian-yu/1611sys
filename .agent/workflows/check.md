# CHECK 工作流：代码完整度检查

本工作流将对照产品规格文档，检查已实现的代码是否完整覆盖所有需求。

## 前置条件

- ✅ 必须存在 `_Product-Spec.md` 文件
- ✅ 项目代码已部分或全部完成

## 执行步骤

1. **读取产品规格文档**

   - 提取所有功能需求（P0/P1/P2）
   - 提取验收标准

2. **扫描项目代码**

   - 检查前端页面文件
   - 检查后端 API 路由
   - 检查数据库 Schema

3. **功能覆盖度检查**
   对于每个需求项：

   - ✅ 已实现
   - ⚠️ 部分实现
   - ❌ 未实现

4. **生成检查报告**
   创建 `_Check-Report.md`：

   ```markdown
   # 代码完整度检查报告

   生成时间：[时间戳]

   ## 总体进度

   - P0 功能：X/Y 已完成 (Z%)
   - P1 功能：X/Y 已完成 (Z%)
   - P2 功能：X/Y 已完成 (Z%)

   ## 详细检查结果

   ### P0 功能

   - [✅] 用户注册登录

     - 前端：`/src/views/auth/Login.vue` ✅
     - 后端：`/src/modules/auth/auth.controller.ts` ✅
     - 数据库：User 表 ✅

   - [⚠️] 仪表盘数据展示

     - 前端：`/src/views/Dashboard.vue` ✅
     - 后端：`/src/modules/dashboard/dashboard.controller.ts` ❌ 缺失
     - 数据库：Dashboard 表 ✅

   - [❌] 数据导出功能
     - 前端：未实现
     - 后端：未实现

   ### P1 功能

   ...

   ## 缺失功能清单

   1. [P0] 仪表盘数据 API 接口
   2. [P0] 数据导出功能（前后端）
   3. [P1] 用户权限管理页面

   ## 建议

   - 优先完成 P0 缺失功能
   - 检查已实现功能的错误处理
   - 添加单元测试覆盖核心逻辑
   ```

5. **输出检查结果**
   - 向用户展示报告
   - 高亮显示缺失的关键功能

## 输出物

- `_Check-Report.md`：代码完整度检查报告
