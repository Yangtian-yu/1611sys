# HELP 工作流：指令帮助

显示所有可用的工作流指令及其说明。

## 可用指令

### 📋 需求与设计

**`/prd`** - 需求收集，生成产品规格文档

- 激活毒舌产品经理技能
- 深度追问需求细节
- 生成 `_Product-Spec.md`

**`/ui`** - 生成 UI 原型图提示词

- 前置条件：已有 `_Product-Spec.md`
- 激活 UI 提示词设计师技能
- 生成 `UI-Prompts.md` 和原型图

---

### 💻 开发与检查

**`/dev`** - 开始开发项目代码

- 前置条件：已有 `_Product-Spec.md`
- 激活全栈开发工程师技能
- 按优先级实现功能（P0 → P1 → P2）

**`/check`** - 检查代码完整度

- 对照 `_Product-Spec.md` 检查实现情况
- 生成 `_Check-Report.md`
- 列出缺失功能清单

---

### 🚀 运行与管理

**`/run`** - 本地运行项目

- 自动安装依赖（pnpm install）
- 启动数据库（如果有 docker-compose）
- 启动前后端服务

**`/stop`** - 停止运行中的服务

- 停止前端服务
- 停止后端服务
- 停止数据库容器（可选）

**`/status`** - 显示项目状态

- 文档完成情况
- 代码统计信息
- 服务运行状态
- 开发进度（P0/P1/P2）

---

### ℹ️ 其他

**`/help`** - 显示此帮助信息

---

## 使用示例

```
典型开发流程：

1. /prd          # 收集需求，生成产品规格文档
2. /ui           # 设计 UI 原型图
3. /dev          # 开始开发
4. /run          # 本地运行测试
5. /check        # 检查完整度
6. /status       # 查看进度
7. /stop         # 停止服务
```

## 注意事项

- 带有 `// turbo` 注释的步骤会自动执行，无需用户确认
- 工作流之间有依赖关系，请按顺序执行
- 所有生成的文档都会保存在项目根目录
